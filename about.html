iSpeak

← Zurück zur App Anleitung
iSpeak ist eine barrierefreie Kommunikationshilfe für Menschen, die ihre Stimme verloren haben oder ihre Hände nicht mehr einsetzen können – steuerbar allein durch Augenbewegungen und Blinzeln.

Was ist iSpeak?

iSpeak ist eine quelloffene Web-Anwendung zur unterstützten Kommunikation (AAC – Augmentative and Alternative Communication). Sie läuft vollständig im Browser, benötigt keine Installation und speichert alle Daten lokal auf dem Gerät – es werden keine persönlichen Daten an Server übertragen.

Die App richtet sich an Menschen mit schweren motorischen Einschränkungen, etwa durch ALS (Amyotrophe Lateralsklerose), Locked-in-Syndrom, Schlaganfall oder ähnliche Erkrankungen. Mit einer einfachen Frontkamera lässt sich die gesamte Oberfläche ohne Hände bedienen.

Wie funktioniert die Steuerung?

iSpeak nutzt die Kamera des Geräts, um die Augen in Echtzeit zu verfolgen (Eye-Tracking via MediaPipe FaceMesh). Die Bedienung erfolgt über drei Gesten:

  • Blick nach oben – scrollt die Auswahl nach oben
  • Blick nach unten – scrollt die Auswahl nach unten
  • Blinzeln – bestätigt die aktuell angezeigte Option

Zum Wechsel zwischen den Modi hält man das Auge für ca. 3 Sekunden geschlossen. Ein Kalibrierungsassistent hilft dabei, das System auf die individuellen Augenbewegungen abzustimmen.

Was kann iSpeak?

  • Buchstabieren – Wörter und Sätze über eine T9-ähnliche Buchstabenauswahl zusammenstellen
  • Wortbaum – vordefinierte Wörter nach Kategorien (Verben, Nomen, Adjektive u. a.) schnell einfügen
  • Sätze – gespeicherte Phrasen aus einer eigenen Bibliothek abrufen
  • Sprechen – den zusammengestellten Text per Text-to-Speech vorlesen lassen
  • Telegram – Nachrichten direkt an Kontakte senden und empfangen
  • Verwalten – Wörter und Sätze in Kategorien organisieren, umbenennen und verschieben

Wo werden die Daten gespeichert?

Alle Wörter, Sätze, Kategorien und Einstellungen werden ausschließlich im lokalen Speicher des Browsers (localStorage) gespeichert. Es findet keine Übertragung an externe Server statt. Die Kamerabilder werden ausschließlich lokal im Browser verarbeitet und nie gespeichert oder übertragen.

Die Verwaltung der gespeicherten Inhalte ist über Speicher verwalten möglich.

Technologie

  • Reines HTML, CSS und JavaScript – keine Frameworks, keine Abhängigkeiten außer MediaPipe
  • Eye-Tracking: Google MediaPipe FaceMesh
  • Text-to-Speech: Web Speech API des Browsers
  • Messaging: Telegram Bot API (optional, erfordert eigenen Bot-Token)
